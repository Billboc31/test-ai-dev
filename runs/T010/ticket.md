# T010 — T010 - Establish project foundation and baseline documentation

**Source**: GitHub Issue #11

## Description

## Goal

Establish the project foundation before implementing product features.

## Scope

- Confirm the product vision for a calendar-based task management app.
- Create or update baseline documentation in `docs/` and `docs/ai/`.
- Document the validated technical stack.
- Define the initial folder structure.
- Add local development instructions.

## Constraints

- No product feature implementation.
- No UI screens beyond minimal placeholders if needed.
- Keep the app simple, monolithic, and responsive.

## Acceptance criteria

- README explains how to run the project locally.
- `docs/ai/global-context.md` contains the project vision and agent context.
- Architecture decisions are documented.
- The repository structure is clear for future tickets.
- CI/lint/test expectations are documented.
